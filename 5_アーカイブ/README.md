# アーカイブ

現行運用では使わないが、履歴として残す資料を置く。

## 重要ルール

- 現行仕様・現行手順として参照しない。
- 新しい作業を始めるときは、`6_システム/LINE_BOT_仕様.md` と `6_システム/開発フロー.md` を優先する。
- アーカイブ内の手順を実行する場合は、必ず現行仕様と照合してから使う。

## GitHub管理の考え方

- READMEや索引はGitHub管理してよい。
- 古い手順書、引き継ぎ、未分類メモは必要になった時だけ個別に確認する。
- APIキー例、旧VPS手順、古いBot手順が混ざる可能性があるため、一括commitしない。
- 現行へ戻す時は、まず `6_システム/FLATUPGYM_AI_整理台帳.md` に昇格理由を書く。
