## 3. 現在の問題（ハルシネーション）

### 症状
- **状況：** JINが手動でLINE返信対応中なのに、Difyが勝手に余計な回答を出力している
- **現象：** 不正確・不要な情報（ハルシネーション）が含まれた回答が送信される
- **影響：** 顧客対応の品質低下、信頼損失のリスク

### 原因の可能性（推定）
1. **LLMモデルの精度が低い**
   - 現在：Mistral 7B（OpenRouter経由）
   - 問題：軽量モデルのため幻覚が多い

2. **RAG（AnythingLLMのベクトル検索）がズレている**
   - 不正確・古い情報がドキュメントに混入している可能性
   - 検索結果の関連性が低い

3. **システムプロンプト（SOUL.md / AIKAペルソナ）が曖昧**
   - 指示が不足または矛盾している
   - 「回答していい範囲」の境界が不明確

4. **温度設定（temperature）が高すぎる**
   - Difyの設定で temperature が 0.7～1.0 だと、回答が「創意的」になりすぎる
   - 推奨値：0.3～0.5（ファクチュアル重視）

## 5. 実施内容：Dify停止手順（成功した方法）

### 環境
- **サーバー：** Xserver VPS（IP: 162.43.90.71）
- **OS：** Ubuntu 25.04
- **稼働中のコンテナ：** 12個（Dify関連 9個 + AnythingLLM + nginx + squid）

### 問題発生の経緯
1. `docker-compose down` → docker-compose.ymlが見つからない
2. `docker stop <container>` → permission denied
3. `sudo docker stop <container>` → permission denied（sudoでも効かない）
4. `docker kill $(docker ps -q)` → permission denied
5. **解決策：OSレベルの再起動**

### 最終的な成功手法
```bash
# VPSへSSH接続
ssh root@162.43.90.71

# OS全体を再起動（Dockerコンテナはリセット）
reboot
```

**結果：**
- VPSが再起動
- Difyコンテナ 12個がすべて停止
- `restart: always` 設定がない場合、再起動後も自動起動しない
- **現在：手動対応モードで稼働**

## 6. 次のフェーズ（後処理）

### 確認事項（VPS再接続後）
```bash
ssh root@162.43.90.71
docker ps
```
- Difyコンテナが起動していないことを確認
- AnythingLLM（port 3001）のみ起動している可能性がある

### 再開時の改善案

#### A. システムプロンプト修正
**SOUL.md / AIKAペルソナの厳密化**
- 「回答していい範囲」を明確に定義
- 不確実な情報は「不明なため、別途確認します」と返す
- 女性・子ども・初心者への配慮を明示
- 不潔な男性への毅然とした態度を定義

#### B. LLMモデル変更
```
現在：Mistral 7B（7BパラメータはMac 8GBでは限界）
候補：
  - Claude 3.5-Sonnet（OpenRouter経由）→ 精度最高
  - Qwen2.5 32B（量子化）→ 精度＋効率のバランス
  - Gemma 3 27B → Ollama環境での最適化
```

#### C. Temperature最適化
```
Dify設定で以下に変更：
  temperature: 0.3 ~ 0.5（現在：推定 0.7以上）
  top_p: 0.9
  frequency_penalty: 0.5
```

#### D. AnythingLLMドキュメント監査
- 古い情報・矛盾しているノートを削除
- Obsidianのドキュメント構造を定期同期
- RAG検索の妥当性をスポットチェック
