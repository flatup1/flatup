# FLATUP AI OS 開発フロー (v1.0)

最終更新: 2026-04-29

## このファイルの役割

FLATUP AI OS（AIKA、LINE Bot、関連スキル）の開発・修正・本番反映の安全な手順を定義する。
すべての AI（Claude Code / Codex / 他）と人間が同じルールで動くための共通仕様書。

---

## 役割分担

| 役 | 担当 | 強み | 弱み |
|---|---|---|---|
| 実装の手 | Claude Code | 設計・実装・コーディング | 自分の作品を前向きに評価しがち |
| 検査官（反対尋問役） | Codex `/codex:adversarial-review` | 本番事故パターンを疑う | 修正はしない |
| 修正の手 | Claude Code（指摘を戻して再実装） | 指摘ベースで的確な修正 | 自己評価は甘い |
| 最終承認者 | JIN | ブランド・倫理・本番判断 | - |
| 本番応答 | AIKA（Claude Haiku） | 会員対応 | データ外のことは答えない |

---

## 開発ループ（必ずこの順）

```
1. Claude Code で実装
       ↓
2. /codex:adversarial-review --base main <疑うべき領域>
       ↓
3. 指摘を Claude Code に戻して修正
       ↓
4. /codex:review --base main で最終確認
       ↓
5. JIN 承認
       ↓
6. 本番反映
```

---

## adversarial-review を必ず挟む領域

これらは「事故ったら会員さんに直接被害が出る」領域。**自動反映禁止、必ず adversarial-review を挟む**。

- AIKA 応答境界（料金・時間・キャンペーン条件のハルシネーション）
- PII / 個人情報フロー（ログ・LINE 履歴・API キー）
- 体験予約導線（女性希望・男性担当の取り違え、不在日誤案内、確定前発言）
- uizin 決済管理（親名義 ↔ 選手名の紐付け、未決済催促重複）
- 退会・休会案内（ルール誤案内、AIKA の言い切り）
- スポンサー管理
- クチコミ催促導線
- 入会手続き自動化

## 標準テンプレ集

### 普通のレビュー（差分の品質チェック）

```
/codex:review --base main
```

### 疑り深いレビュー（汎用・最強テンプレ）

```
/codex:adversarial-review --base main \
  challenge whether this design can fail in production due to \
  race conditions, partial failures, wrong assumptions, \
  privacy leaks, or unsafe automation
```

意味: この設計が本番環境で、同時処理・部分失敗・前提ミス・個人情報漏洩・危険な自動化で破綻しないか、厳しくレビューして

### LINE Bot 用テンプレ

```
/codex:adversarial-review --base main \
  challenge whether this LINE reply flow can hallucinate \
  prices, schedules, or campaign conditions
```

### 個人情報用テンプレ

```
/codex:adversarial-review --base main \
  challenge privacy risks and data leakage in this customer CRM flow
```

### 予約導線用テンプレ

```
/codex:adversarial-review --base main \
  challenge whether this trial booking flow prevents double booking \
  and wrong instructor availability
```

### uizin 決済用テンプレ

```
/codex:adversarial-review --base main \
  challenge whether this payment tracking flow can distinguish \
  paid and unpaid fighters without ambiguity
```

## 自動反映の禁止リスト（絶対 AI 単独でやらない）

以下は **JIN 承認必須**:

- API キー / 認証情報の変更
- AIKA の人格プロンプト（`1_AIKA人格_本番.md`）の変更
- LINE Bot の応答仕様変更（line_webhook.py の応答ロジック）
- 会員データへの書き込み
- Git の履歴改変・force push
- 本番サービスの再起動（障害対応を除く）
- 環境変数（`/root/flatup_config.env`）の変更

---

## 自動許可の範囲（AI が自律で実行 OK）

- サービス再起動（既知の障害発生時のみ）
- ファイル権限の自動修正（700/600 への戻し）
- ログローテーション
- 古いログの自動削除（180 日超）
- 改善案の生成（**生成のみ、反映は禁止**）
- 監視チェック・アラート発火

## 本番アーキテクチャ原則

```
LINEユーザー
    ↓
AIKA本番Bot (/root/line_webhook.py)
    ↓
PIIマスク済みログ (Obsidian Vault LOGS/)

別系統:
Obsidian Vault
    ↓
Claude Code / Codex / 他エージェント
    ↓
改善案・投稿案・デザイン案
    ↓
JIN承認
    ↓
本番反映
```

**鉄則**: 本番 AIKA に外部エージェント（HERMES、OpenClaw、その他）を直結しない。改善案は提案までを自動、本番反映は必ず JIN を経由する。

## Codex コマンドの使い分け

| コマンド | 用途 | 修正する？ |
|---|---|---:|
| `/codex:review` | 通常のコードレビュー | しない |
| `/codex:adversarial-review` | 反対尋問・本番事故テスト | しない |
| `/codex:rescue` | バグ調査・修正委任（公式） | する |

→ `adversarial-review` は **review-only**。指摘が出たら Claude Code に戻して修正させる。
