## 昇格基準

学習ログに残すのは、次回のAIが読んだ時に判断が良くなるものだけです。

| 残す | 残さない |
|---|---|
| JINが明確に決めた運用ルール | まだ迷っている案 |
| 同じミスを防ぐ注意点 | その場だけの雑談 |
| APIキー・個人情報を守るルール | 秘密情報そのもの |
| 本番反映や投稿公開の安全ルール | AIの思いつき |
| 使い回せるコマンド・手順 | 一度だけの作業メモ |

迷ったら、まず `4_日記/JIN_AI_INBOX.md` に置き、JIN承認後にこの学習ログへ昇格します。

## 2026-05-17 Typefully連携はAPIキー本体を書かずに管理する

### 起きたこと

- Typefully APIはローカルPCから疎通確認できた。
- `flatupgym` の social set ID は `307036`。
- APIキーは `/Users/jin/.config/typefully/config.json` に保存する。
- Obsidian、README、GitHubにはAPIキー本体を書かない。

### 学んだこと

- 投稿自動化は便利だが、公開までAIに任せると危険。
- X投稿はTypefullyで下書き保存し、人間が確認してから公開・予約するのが安全。
- 1日5投稿、オリジナル比率70%以上、自動リプライ禁止を明文化する必要がある。

### 次からのルール

- Typefully関連の説明にはキー本体を書かない。
- AIは `--immediate` や即時公開操作を勝手に使わない。
- 参考ポストを使う場合も、FLATUPの経験と言葉に置き換える。

### 反映先

- `02_SKILLS_MARKETING/SNS/Typefully_X運用メモ_FLATUP.md`

## 2026-05-17 100点化は「説明書」と「整理台帳」で進める

### 起きたこと

- JINから「100点に近づけて整理して最適化して中学生にもわかるように説明書作って」と依頼があった。
- Vault内には未整理ファイルや未コミット差分が多く、いきなり移動すると本番影響や混乱のリスクがある。

### 学んだこと

- 100点化は、ファイルを一気に動かすことではない。
- まず誰でも理解できる説明書を作り、未整理ファイルを安全に分類する台帳を作るのが先。
- 中学生向けに説明できない仕組みは、運用でも迷いやすい。

### 次からのルール

- 未整理ファイルは、移動前に「現行」「保留」「アーカイブ」「削除候補」の分類案を作る。
- 本番コード、休業日設定、AIKA人格は、分類作業から分離する。
- 説明は、Obsidian=脳、AIOS=手足、GitHub=タイムマシンの比喩で統一する。

### 反映先

- `6_システム/FLATUPGYM_AI_中学生向け説明書.md`
- `6_システム/FLATUPGYM_AI_整理台帳.md`
- `00_CORE/FLATUPGYM_AI_HOME.md`

## 2026-05-17 100点化は「分類」と「本番差分レビュー」で到達

### 起きたこと

- 97点時点では、未整理ファイル分類と本番コード差分レビューが残っていた。
- `closed_dates.json` にJSON構文エラーが見つかった。
- 本番コード差分には、owner専用コマンド、動画、リサーチ、PII外部化など大きな機能追加が含まれていた。

### 学んだこと

- 100点化とは、すべてを本番反映することではない。
- 危険な差分を「不明な状態」から「レビュー済みの保留差分」に変えることが、AI母艦としての100点。
- 本番コード差分は、整理台帳とは別に安全レビューを残すべき。

### 次からのルール

- JSONや設定ファイルは、commit前に構文確認する。
- 本番コードに関わる差分は、仕様書・レビュー・テスト・JIN承認を分ける。
- 大きな未追跡ファイル群は、移動前に台帳で分類する。

### 反映先

- `6_システム/FLATUPGYM_AI_整理台帳.md`

## 2026-05-17 最適化では「危険なローカル設定」と「余計なGit履歴」を先に消す

### 起きたこと

- JINから「最適化して不要なものは削除して」と依頼があった。
- `.claude/settings.local.json` にローカル実行許可とVPS接続情報が含まれていた。
- `99_REFERENCE/exbrain/.git/` がVault内に残っており、親リポジトリのGit管理と混ざる状態だった。

### 学んだこと

- 不要ファイル削除では、内容のあるメモより先に、秘密情報・ローカル設定・別リポジトリの管理情報を片づける。
- 参考資料そのものは残してよいが、参考資料内の `.git/` はObsidian Vaultの知識管理には不要。
- `.claude/settings.local.json` は便利でもGitHub管理対象にしてはいけない。

### 次からのルール

- `.claude/settings.local.json` と `*.local.json` は `.gitignore` で無視する。
- Vault内に別リポジトリを置く場合、`.git/` は削除するか、参考資料として完全に管理外に置く。
- 削除は、重複・危険・機械生成メタデータに限定し、JINの判断や知識が入ったメモは勝手に消さない。

### 反映先

- `.gitignore`
- `6_システム/FLATUPGYM_AI_整理台帳.md`
- `4_日記/FLATUPGYM_AI_学習ログ.md`
