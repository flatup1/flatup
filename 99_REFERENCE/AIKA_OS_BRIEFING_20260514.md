## 7. 大事な開発原則

1. **既存システムを壊さない**（auto 経路は完全維持）
2. **提案 → 承認 → 反映**（JIN の決定権を奪わない）
3. **adversarial-review を挟む**(Codex に疑り深くレビューさせる)
4. **fail-safe**（曖昧時は安全側 = approval に倒す）
5. **コマンドは OWNER 限定**（OWNER_USER_ID 以外が打っても無効）
6. **小さく始めて、確認しながら積み上げる**（YAGNI）
7. **本番反映前に必ず:**
   - バックアップ
   - 構文チェック（`python -m py_compile`）
   - import 確認
   - JSON 妥当性
   - `/health` で 200 OK
   - LINE 実機テスト
