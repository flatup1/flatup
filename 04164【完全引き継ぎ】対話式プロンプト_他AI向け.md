```
【取得方法】

ステップ1: LINE 管理画面にログイン
└─ URL: https://manager.line.biz/

├─ メニュー: 左パネル「チャネル設定」
├─ ページ: 「基本情報」
├─ 項目: 「チャネルID」をコピー
└─ 例: 1234567890

ステップ3: チャネルシークレット を取得
├─ メニュー: 同じ「基本情報」ページ
├─ 項目: 「チャネルシークレット」をコピー
└─ 例: abc123def456ghi789...

ステップ4: アクセストークン を取得
├─ メニュー: 左パネル「アカウント設定」
├─ ページ: 「API」または「Messaging API」
├─ 項目: 「アクセストークン」をコピー
│        （「Bearer」で始まる長い文字列）
└─ 例: Bearer Cxxx...

【注意】
・これら 3つは絶対に外部に漏らさない（秘密鍵）
・.env.local には記述するが、GitHub には commit しない
├─ .gitignore に「.env.local」を追加して確認
└─ git config user.email / user.name も確認

【Webhook URL の設定】
セットアップ後、LINE 管理画面で以下を設定:
├─ 「Messaging API」→「Webhook URL」
├─ URL: http://localhost:3000/line/webhook （開発時）
│       https://[VPS IP or domain]/line/webhook （本番時）
└─ 「Webhook の使用」を「有効」に設定
```

## 🎯 Q14: 失敗時のトラブルシューティングは？

**A14: よくあるエラーと対処法**

```
【エラー 1: Ollama に接続できない】
症状: Error: Failed to connect to http://localhost:11434
原因: Ollama が起動していない
対処:
  1. 別ターミナルで `ollama serve` を実行
  2. Gemma 2 が入っているか確認: `ollama list`
  3. 再度 `npm start` を実行

【エラー 2: SOUL.md が見つからない】
症状: Error: ENOENT: no such file or directory 'config/SOUL.md'
原因: SOUL.md がコピーされていない、または パスが違う
対処:
  1. ファイルの存在確認: `ls -la ~/GitHub/exbrain/config/SOUL.md`
  2. なければ再コピー: `cp ~/Downloads/SOUL.md ~/GitHub/exbrain/config/SOUL.md`
  3. .env.local の SOUL_FILE_PATH を確認

【エラー 3: LINE メッセージが返ってこない】
症状: LINE テストメッセージ送信 → 返答なし
原因: 複数の可能性
対処:
  1. .env.local に LINE_CHANNEL_ID, SECRET, TOKEN が正しく入っているか確認
     `cat .env.local | grep LINE_`
  2. npm start のログで「✓ Listening for LINE messages...」が表示されているか確認
  3. LINE 管理画面で Webhook が有効か確認
  4. ファイアウォール・ネットワーク設定を確認

【エラー 4: npm install でエラー】
症状: npm ERR! code ERESOLVE または npm WARN deprecated
原因: Node バージョンまたは依存関係の競合
対処:
  1. Node バージョン確認: `node -v` （18以上必須）
  2. npm キャッシュをリセット: `npm cache clean --force`
  3. 再インストール: `npm install --legacy-peer-deps`

【エラー 5: [ユーザー名] の部分がわからない】
症状: .env.local の SOUL_FILE_PATH が「パス不正」エラー
原因: [ユーザー名] が whoami の出力と一致していない
対処:
  1. ターミナルで `whoami` を実行
  2. 出力された名前（例：jin）を .env.local に入力
  3. 例: `/Users/jin/Library/Mobile\ Documents/...`

【デバッグモードで詳細ログを見る】
.env.local に以下を追加:
DEBUG=true
LOG_LEVEL=debug

その後 `npm start` で詳細なログが表示される
```
