# CUSTOMER SKILLS

LINE返信、体験予約、成約率改善、FAQなど、顧客対応の型を置く。

現行Botの仕様判断は `6_システム/LINE_BOT_仕様.md` を優先する。

## 正本資料

| ファイル | 使い道 |
|---|---|
| `line_reply.md` | LINE/DM返信の基本テンプレート |
| `trial_booking.md` | 体験予約案内の型 |
| `【FLATUP AI OS】成約特化型ナレッジFAQリスト.md` | FAQ・反論処理・案内文の元ネタ |
| `FLATUPGYM 成約率最大化マスターガイド：体験設計とLINE接客アルゴリズム.md` | 体験導線と接客改善の設計資料 |

## 使う時のルール

- 予約確定、料金変更、休業日判断はAIだけで決めない。
- 会員や体験者の個人情報はGitHubに残さない。
- 返信文は、AIKA人格とFLATUPの安心感に合わせる。
- 迷ったら `6_システム/LINE_BOT_仕様.md` と `1_AIKA人格_本番.md` を優先する。
